<h2><a id="user-content-binary-downloads" class="anchor" aria-hidden="true" href="#binary-downloads"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Binary Downloads</h2>

<h2><a id="user-content-main-features" class="anchor" aria-hidden="true" href="#main-features"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Main Features</h2>

<h2><a id="user-content-installation" class="anchor" aria-hidden="true" href="#installation"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Installation</h2>
<p>Download and install <a href="https://dl.nwjs.io/v0.14.7" rel="nofollow">NW.js v0.14.7 LTS</a> and <a href="https://nodejs.org/download/release/v5.12.0/" rel="nofollow">Node.js v5.12.0</a>.  These versions are recommended for easiest install but newer versions will work too.  If you already have another version of Node.js installed, you can use <a href="https://github.com/creationix/nvm">NVM</a> to keep both.</p>
<p>Clone the source:</p>
<div class="highlight highlight-source-shell"><pre>git clone https://github.com/byteball/byteball.git
<span class="pl-c1">cd</span> byteball</pre></div>
<p>If you are building for testnet, switch to testnet branch:</p>
<div class="highlight highlight-source-shell"><pre>git checkout testnet</pre></div>
<p>Install <a href="http://bower.io/" rel="nofollow">bower</a> and <a href="http://gruntjs.com/getting-started" rel="nofollow">grunt</a> if you haven't already:</p>
<div class="highlight highlight-source-shell"><pre>npm install -g bower
npm install -g grunt-cli</pre></div>
<p>Build Byteball:</p>
<div class="highlight highlight-source-shell"><pre>bower install
npm install
grunt</pre></div>
<p>If you are on Windows or using NW.js and Node.js versions other than recommended, see <a href="http://docs.nwjs.io/en/latest/For%20Users/Advanced/Use%20Native%20Node%20Modules/" rel="nofollow">NW.js instructions about building native modules</a>.</p>
<p>After first run, you'll likely encounter runtime error complaining about node_sqlite3.node not being found, copy the file from the neighboring directory to where the program tries to find it, and run again. (e.g. from <code>byteball/node_modules/sqlite3/lib/binding/node-v47-darwin-x64</code> to <code>byteball/node_modules/sqlite3/lib/binding/node-webkit-v0.14.7-darwin-x64</code>)</p>
<p>Then run Byteball desktop client:</p>
<div class="highlight highlight-source-shell"><pre>/path/to/your/nwjs/nwjs <span class="pl-c1">.</span></pre></div>

<h3><a id="user-content-ios" class="anchor" aria-hidden="true" href="#ios"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>iOS</h3>

</ul>
</li>
</ul>

<h2><a id="user-content-byteball-backups-and-recovery" class="anchor" aria-hidden="true" href="#byteball-backups-and-recovery"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Byteball Backups and Recovery</h2>
<p>Byteball uses a single extended private key for all wallets, BIP44 is used for wallet address derivation.  There is a BIP39 mnemonic for backing up the wallet key, but it is not enough.  Private payments and co-signers of multisig wallets are stored only in the app's data directory, which you have to back up manually:</p>
<ul>
<li>macOS: <code>~/Library/Application Support/byteball</code></li>
<li>Linux: <code>~/.config/byteball</code></li>
<li>Windows: <code>%LOCALAPPDATA%\byteball</code></li>
</ul>
<h2><a id="user-content-translations" class="anchor" aria-hidden="true" href="#translations"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Translations</h2>
<p>Byteball uses standard gettext PO files for translations and <a href="https://crowdin.com/project/byteball" rel="nofollow">Crowdin</a> as the front-end tool for translators. To join our team of translators, please create an account at <a href="https://crowdin.com" rel="nofollow">Crowdin</a> and translate the Byteball documentation and application text into your native language.</p>
<p>To download and build using the latest translations from Crowdin, please use the following commands:</p>
<div class="highlight highlight-source-shell"><pre><span class="pl-c1">cd</span> i18n
node crowdin_download.js</pre></div>
<p>This will download all partial and complete language translations while also cleaning out any untranslated ones.</p>
